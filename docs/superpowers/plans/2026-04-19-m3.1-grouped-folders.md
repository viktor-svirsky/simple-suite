# M3.1 Grouped Folders View Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the single-folder filter view ("select Work → show only Work notes") with an all-folders-at-once grouped view (Todoist-style): each folder renders its own section with a header and a per-section `+ Add note` affordance. Notes with no folder go into an "Inbox" section at top.

**Architecture:**
- `NoteListView` gains an `isGrouped: Bool` toggle (default on). When on, it skips the folder predicate and instead fetches **all** visible notes, groups them in-memory by `folder?.name`, and renders a `ForEach` over `FolderSection` entries. Each section header has a `+` that creates a note with `folder = sectionFolder`.
- Search keeps working by narrowing the pre-grouped fetch.
- Sidebar folder rows become "scroll-to" targets instead of filters — tapping a folder scrolls the grouped list to that section via `ScrollViewReader`.

**Tech Stack:** SwiftUI `List` + `Section`, `ScrollViewReader`, SwiftData `@Query`.

**Branch:** `feat/m3.1-grouped-folders` off `main`.

## Scope

### In (M3.1)
- Grouped list rendering: `Inbox` section (no folder) + one section per folder
- Folders ordered by `sortOrder` asc, then name
- Per-section `+` button creates a note assigned to that folder (or none, for Inbox)
- Section count badge (like Todoist)
- Sidebar folder tap → scroll to section (not filter)
- Swipe-to-delete and swipe-to-pin still work per row
- Pinned notes float to the top of their folder's section (not a separate section) — spec stays consistent
- Search narrows within groups (sections that become empty disappear)

### Out
- Drag-to-reorder notes or folders (later)
- Collapsible sections (later)
- Compact/card density toggle

final class NoteListGroupingTests: XCTestCase {
    func test_groups_notesByFolder_preservingInboxFirst() {
        let journal = Folder(name: "Journal", sortOrder: 0)
        let work = Folder(name: "Work", sortOrder: 1)
        let a = Note(body: "a")
        let b = Note(body: "b"); b.folder = journal
        let c = Note(body: "c"); c.folder = work
        let d = Note(body: "d"); d.folder = work

        let sections = NoteListGrouping.group([a, b, c, d])

        XCTAssertEqual(sections.count, 3)
        XCTAssertNil(sections[0].folder)             // Inbox
        XCTAssertEqual(sections[0].notes.map(\.body), ["a"])
        XCTAssertEqual(sections[1].folder?.name, "Journal")
        XCTAssertEqual(sections[2].folder?.name, "Work")
        XCTAssertEqual(sections[2].notes.map(\.body), ["c", "d"])
    }

    func test_ordering_foldersBySortOrder() {

```swift
import Foundation

struct FolderSection: Identifiable {
    var id: UUID? { folder?.id }
    let folder: Folder?
    let notes: [Note]

    var title: String { folder?.name ?? "Inbox" }
}

enum NoteListGrouping {
    static func group(_ notes: [Note]) -> [FolderSection] {
        var byFolder: [UUID: [Note]] = [:]
        var inbox: [Note] = []
        var foldersByID: [UUID: Folder] = [:]

        for note in notes {
            if let f = note.folder {
                byFolder[f.id, default: []].append(note)
                foldersByID[f.id] = f
            } else {
                inbox.append(note)
            }
        }

        var sections: [FolderSection] = []
        if !inbox.isEmpty {
            sections.append(FolderSection(folder: nil, notes: inbox))
        }
        for folder in foldersByID.values.sorted(by: { ($0.sortOrder, $0.name) < ($1.sortOrder, $1.name) }) {
            let notes = byFolder[folder.id] ?? []
            sections.append(FolderSection(folder: folder, notes: notes))
        }
        return sections
    }
}
```
